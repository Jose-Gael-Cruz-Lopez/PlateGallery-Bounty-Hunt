# PlateGallery Backend

Two Python services that power the PlateGallery API.

---

## Services

### `api/` — Main REST API (port 4000)
FastAPI app that serves the frontend with plate data, leaderboard rankings, and state statistics.

### `moderation/` — Moderation Service (port 8001)
FastAPI app that receives moderation jobs when a plate is submitted. Currently a starter pipeline that accepts everything — built to be extended with real image checks.

---

## What the API does right now

The API runs entirely off **in-memory seed data** (no database connected yet). It supports:

### Plates
- `GET /api/plates` — Returns a paginated list of approved plates. Supports filtering by `state` and sorting by `newest`, `top` (Wilson score), or `trending` (hot score).
- `POST /api/plates` — Accepts a new plate submission (`state`, `plateText`, `description`). Creates a UUID, sets status to `pending`, and returns a moderation job payload. Nothing is persisted yet.
- `GET /api/plates/{id}/status` — Returns the moderation status for a plate. Always returns `pending` for now.
- `POST /api/plates/{id}/vote` — Accepts an upvote (`1`) or downvote (`-1`). Returns the value back but does not update any scores yet.

### Leaderboard
- `GET /api/leaderboard` — Returns the top 10 plates sorted by Wilson score. Optionally filtered by `state`.

### States
- `GET /api/states` — Returns plate count, contributor count, and latest upload date for each state. Currently covers 12 states from seed data.

### Health
- `GET /health` — Returns `{ ok: true, service: "api" }`.

---

## What is NOT wired up yet

- **Database** — Supabase credentials are in `.env` but no queries are made. All data comes from `seed_data.py`.
- **Persistent votes** — Votes are acknowledged but scores are not updated.
- **Image uploads** — The `POST /api/plates` endpoint accepts metadata only. No file upload or Cloudinary integration yet.
- **Real moderation** — The moderation service always approves. Extension points are in `moderation/app/pipeline.py` for OpenAI / Google Vision checks.
- **Auth** — Clerk keys are in `.env` but no auth middleware is applied to any route.

## Running locally

**API:**
```bash
cd backend/api
pip install -r requirements.txt
python main.py
# → http://localhost:4000
```

**Moderation service:**
```bash
cd backend/moderation
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8001
# → http://localhost:8001
```

Interactive API docs (auto-generated by FastAPI) are available at `http://localhost:4000/docs`.

## Environment variables

Copy `.env.example` at the repo root and fill in values as services are wired up.

| Variable | Used by | Purpose |
|---|---|---|
| `PORT` | api | API server port (default 4000) |
| `SUPABASE_URL` | api | Database (not wired yet) |
| `SUPABASE_SERVICE_ROLE_KEY` | api | Database admin key (not wired yet) |
| `CLOUDINARY_CLOUD_NAME` | api | Image hosting (not wired yet) |
| `CLERK_SECRET_KEY` | api | Auth verification (not wired yet) |
| `MODERATION_SERVICE_URL` | api | URL of the moderation service |
| `OPENAI_API_KEY` | moderation | GPT-4o moderation pipeline (not wired yet) |
